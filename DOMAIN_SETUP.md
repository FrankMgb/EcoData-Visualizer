# Domain Configuration Guide (Cloudflare + AWS Amplify)

This guide documents the process of connecting a domain managed by **Cloudflare** (`climateflows.com`) to an **AWS Amplify** application.

## 1. Prerequisites
-   A domain name purchased/managed on [Cloudflare](https://www.cloudflare.com/).
-   An AWS Amplify app provisioned (via Terraform or Console).

## 2. AWS Configuration (Terraform)
We use Terraform to tell AWS Amplify to associate the domain with our app.

**File:** `terraform/amplify.tf`
```hcl
resource "aws_amplify_domain_association" "domain" {
  app_id      = aws_amplify_app.climate_flow.id
  domain_name = "climateflows.com"
  wait_for_verification = false  # Important: Prevents Terraform from hanging during SSL checks

  sub_domain {
    branch_name = aws_amplify_branch.main.branch_name
    prefix      = ""
  }

  sub_domain {
    branch_name = aws_amplify_branch.main.branch_name
    prefix      = "www"
  }
}
```
Run `terraform apply` to provision this resource. AWS will generate a verification CNAME record.

## 3. SSL Verification (Crucial Step)
AWS needs to verify domain ownership to issue a free SSL certificate.

1.  **Retrieve Record:** These values are **generated by AWS** when you create the domain association. You find them by running:
    ```bash
    aws amplify get-domain-association --app-id <YOUR_APP_ID> --domain-name <YOUR_DOMAIN>
    ```
    *Look for `certificateVerificationDNSRecord` in the JSON output.*
2.  **Add to Cloudflare:**
    -   **Type:** `CNAME`
    -   **Name:** `_5a60aab8...` (The part before .climateflows.com)
    -   **Target:** `_7ffa3f...acm-validations.aws`
    -   **Proxy Status:** ⚠️ **DNS Only** (Grey Cloud)

> [!IMPORTANT]
> You **MUST** set Proxy Status to **DNS Only** (Grey Cloud) for this specific record. If it is Proxied (Orange Cloud), AWS cannot see the record and validation will fail.

## 4. Routing Traffic (Frontend)
Once SSL is verified, route the actual user traffic to your app.

1.  **Get CloudFront URL:** This is also in the output of the CLI command above (under `subDomains` -> `dnsRecord`), or visible in the Amplify Console under "Hosting > Custom domains".
    *It will look like `d3ftoy6eiv4h1a.cloudfront.net`.*
2.  **Add to Cloudflare:**
    -   **Root Record:**
        -   **Type:** `CNAME` (NOT A Record)
        -   **Name:** `@`
        -   **Target:** `d3ftoy6eiv4h1a.cloudfront.net`
        -   **Proxy Status:** **Proxied** (Orange Cloud) is recommended for CDN/WAF benefits.
    -   **WWW Record:**
        -   **Type:** `CNAME`
        -   **Name:** `www`
        -   **Target:** `d3ftoy6eiv4h1a.cloudfront.net`
        -   **Proxy Status:** **Proxied** (Orange Cloud).

## 5. Troubleshooting

### Error: "Valid IPv4 address is required"
-   **Cause:** You selected **Type: A** in Cloudflare.
-   **Fix:** Change Type to **CNAME**. AWS CloudFront uses dynamic IPs, so you must point to the domain name, not a static IP.

### Error: AWS Stuck in "Pending Verification"
-   **Cause:** The SSL Verification CNAME is likely "Proxied" (Orange Cloud).
-   **Fix:** Edit the record in Cloudflare and toggle the cloud icon to **Grey (DNS Only)**.
